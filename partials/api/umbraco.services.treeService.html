treeService (service in module umbraco.services )

Description

The tree service factory, used internally by the umbTree and umbTreeItem directives

Methods

  • clearCache(args)

    Clears the tree cache - with optional cacheKey, optional section or optional filter.

    Parameters

    • args – {Object} –

      arguments

    • args.cacheKey – {String} –

      optional cachekey - this is used to clear specific trees in dialogs

    • args.section – {String} –

      optional section alias - clear tree for a given section

    • args.childrenOf – {String} –

      optional parent ID - only clear the cache below a specific node

  • getChildNode(treeNode, id)

    Gets a child node with a given ID, from a specific treeNode

    Parameters

    • treeNode – {object} –

      to retrive child node from

    • id – {int} –

      id of child node

  • getChildren(args)

    Gets the children from the server for a given node

    Parameters

    • args – {object} –

      Arguments

    • args.node – {object} –

      tree node object to retrieve the children for

    • args.section – {string} –

      current section alias

  • getDescendantNode(treeNode, id, treeAlias)

    Gets a descendant node by id

    Parameters

    • treeNode – {object} –

      to retrive descendant node from

    • id – {int} –

      id of descendant node

    • treeAlias – {string} –
      • optional tree alias, if fetching descendant node from a child of a listview document

  • getMenu(args)

    Returns available menu actions for a given tree node

    Parameters

    • args – {object} –

      Arguments

    • args.treeNode – {string} –

      tree node object to retrieve the menu for

  • getPath(node)

    This will return the current node's path by walking up the tree

    Parameters

    • node – {object} –

      Tree node to retrieve path for

  • getTree(args)

    gets the tree, returns a promise

    Parameters

    • args – {object} –

      Arguments

    • args.section – {string} –

      Section alias

    • args.cacheKey – {string} –

      Optional cachekey

  • getTreeAlias(treeNode)

    Gets the node's tree alias, this is done by looking up the meta-data of the current node's root node

    Parameters

    • treeNode – {object} –

      to retrive tree alias from

  • getTreePackageFolder(treeAlias)

    Determines if the current tree is a plugin tree and if so returns the package folder it has declared so we know where to find it's views, otherwise it will just return undefined.

    Parameters

    • treeAlias – {String} –

      The tree alias to check

  • getTreeRoot(treeNode)

    Gets the root node of the current tree type for a given tree node

    Parameters

    • treeNode – {object} –

      to retrive tree root node from

  • loadNodeChildren(args)

    Clears all node children, gets it's up-to-date children from the server and re-assigns them and then returns them in a promise.

    Parameters

    • args – {object} –

      An arguments object

    • args.node – {object} –

      The tree node

    • args.section – {object} –

      The current section

    Parameters
