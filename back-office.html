1. Grid system

The grid system is a design tool that help and achieve visuel unity across Umbraco. The grid system is built on top of Twitter Bootstrap.

2. Section Bar

The section bar on the left side contain a avatar and 5 default sections: Content, Media, Users, Settings, Developer. Contains the bar more than 5 icons an extra icon appears. By clicking this a new section bar appear. Se animation. The avatar picture disappears and instead a back arrow appears.

3. Header

The header contain a minimal of information, to keep it simpel and clear to the user.

Page name, witchs can easily been edit, just by clicking the text. Here you also have the ability to change language. In the right side of the header 2 buttons are showen, Preview and Publish. The preview button opens a new page with the current page. The Publish button is saving and publishing the current page. Umbraco is constantly saving the current document in the background, so you don't need to press save, but you can still access the Save button by clicking the arrow down, besides Publish. The header also contains tabs. The maximum number of tabs is 3. If more than 3 tabs, they will get a drop down. All header will always be visible to the user.

4. Tree

The tree is an hierarchical list of items related (and usually restricted) to a specific concept, which could be something like a content tree or a media tree.

The header of the tree contain a search function where you can easily access all content in Umbraco.

All icons in the tree are generated with a @font-face.

The tree contains tree states. Normal, hover and active. In hover and active state it is posible to access the context menu. The context menu contain the ablity to create a new document and do something else.

4. Typography

For Umbraco we have chosen the font Open Sans. It was optimized for web, mobile interfaces and has excellent legibility characteristics in its letterforms.

Open Sans is a humanist sans serif typeface designed by Steve Matteson, Type Director of Ascender Corp. It contains 897 character set, which includes the standard ISO Latin 1, Latin CE, Greek and Cyrillic character sets. Open Sans was designed with an upright stress, open forms and a neutral, yet friendly appearance. The font is license under Apache License, version 2.

Available in following cuts:

  • Normal font-weight: 400
  • Normal, Italicfont-weight: 400; font-style:italic
  • Semi-bold font-weight: 500
  • Semi-bold, Italicfont-weight: 500; font-style:italic

    Buttons

    Button styles can be applied to anything with the .btn class applied. However, typically you'll want to apply these to only <a> and <button> elements for the best rendering.

    Button class="" Description
    btn Standard gray button with gradient
    btn btn-primary Provides extra visual weight and identifies the primary action in a set of buttons
    btn btn-info Used as an alternative to the default styles
    btn btn-success Indicates a successful or positive action
    btn btn-warning Indicates caution should be taken with this action
    btn btn-danger Indicates a dangerous or potentially negative action
    btn btn-inverse Alternate dark gray button, not tied to a semantic action or use
    btn btn-link Deemphasize a button by making it look like a link while maintaining button behavior
